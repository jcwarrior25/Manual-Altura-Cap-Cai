10. Evaluación Honesta y Recomendaciones

✅ Lo Que Funciona Bien

  • Taxonomía Sólida: El namespace de tags (nw:, ia:, etc.) es consistente y bien pensado.
  • Campos Personalizados Ricos: Se captura buena información cuando se llena (CURPs, grados, notas).
  • Integración IA Activa: El bot califica y escala a humanos de forma inteligente.
  • Asignación Efectiva: 91% de los contactos están asignados correctamente a usuarios.
  • Pipeline Definido: 13 etapas que cubren el viaje completo.

🔴 Lo Que Está Desordenado

  • Rastreo de Fuente Roto: Campo nativo vacío en 72%. Datos esparcidos entre tags y custom fields.
  • Contaminación de Pipeline: Migración mezclada con leads orgánicos actuales.
  • Caos en Campos de Campus: 3 a 4 campos guardando la misma información con distintos formatos.
  • Errores de Codificación: Artefactos en valores de fuente por workflows defectuosos.
  • Ceguera Geográfica: Cero datos de dirección para cruzar con los 5 campus.

🛠️ Plan de Acción / Recomendaciones para Ops

  1. Normalizar fuente al ingreso: Crear un Workflow que mapee tags + origen del form hacia el campo estándar source.
  2. Etiquetar registros migrados: Aplicar tag migrated:true o crm:legacy para poder excluirlos de reportes de performance actual.
  3. Corregir codificación en Workflows: Revisar el trigger/paso que inyecta "Formulario para pag web" por caracteres inválidos.
  4. Consolidar campos de campus: Elegir un campo canónico único, normalizar la data histórica y deprecar/eliminar los duplicados.
  5. Implementar purga de basura: Eliminar periódicamente registros de prueba y SPAM de los pipelines activos.
